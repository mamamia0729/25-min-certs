Day 1: What is IaC & Terraform

25 min read

Focus Timer 25:00

Theory (10 min read)

The problem Terraform solves

Before Terraform, to create a server you clicked around in AWS manually. If you needed 10 servers — you clicked 10 times. If something broke, you didn't know what you had. That's the problem.

IaC = Infrastructure as Code. Instead of clicking, you write a file that describes what you want. Terraform reads that file and builds it for you.

Think of it like a recipe. You write the recipe once. You can cook it 100 times, share it with anyone, and it always makes the same dish.

How Terraform thinks (the most important concept)

Terraform is declarative — you say what you want, not how to build it.

📄
Your file says "I want 2 servers"
☁️
Cloud has 1 server running
✅
Terraform creates +1 more server
Desired state = Real state

You never write step-by-step instructions. You just describe the end result.

The 4 commands you'll use every single day

📥
init Get the tools ready
🔍
plan Preview what changes
🔨
apply Build it for real
🗑
destroy Tear it all down

Rule: Always plan before apply. Always. No exceptions.

3 things every Terraform project has

main.tf         → Where you describe what you want
variables.tf    → The inputs (optional today)
outputs.tf      → The results printed after apply (optional today)

Start with just main.tf. That's all you need today.

What a Terraform file looks like

terraform {
  required_providers {
    local = {
      source = "hashicorp/local"
    }
  }
}

resource "local_file" "hello" {
  filename = "hello.txt"
  content  = "Hello, Terraform!"
}

Break it down:

  • terraform {} → tells Terraform what provider (tool) to use
  • resource → the thing you want to create
  • "local_file" → the type of resource
  • "hello" → your nickname for it (you pick this)
  • filename / content → the settings for that resource

Hands-On (35 min)

Step 1 — Create your folder

mkdir terraform-30days
cd terraform-30days
mkdir day01
cd day01

Step 2 — Create main.tf

Create main.tf and paste the code block above into it.

Step 3 — Run the 4 commands in order

terraform init
terraform plan
terraform apply    # type "yes" when asked

Check your folder — hello.txt appeared.

Step 4 — Modify and re-apply

Change the content in main.tf to anything else, run terraform apply again. Watch it update.

Step 5 — Destroy

terraform destroy  # type "yes"

hello.txt is gone.

Break It On Purpose (15 min)

Try each of these, run terraform plan, read the error:

  1. Delete the quotes around "hello.txt" — what happens?
  2. Rename resource to resources — what happens?
  3. Remove the content line entirely — what happens?

Reading errors without panic is a core Terraform skill. Errors are just Terraform telling you exactly what's wrong.

What You Learned Today

  • IaC means writing files instead of clicking
  • Terraform is declarative — describe the end result
  • init → plan → apply → destroy is the core loop
  • A resource block has: type, name, and settings
  • Errors are readable and fixable
